7. CONCLUSION


Challenges

- Slow Internet connection speed results in elements not loading properly.
- XPATH and Full XPATH values for some web elements are dynamic.
- Cannot detect when a file has finished from being uploaded to GitHub. By default, the program is set to wait for a maximum of 10 seconds before uploading the next file.

Lessons Learned

- Tkinter, Selenium, GitHub Pages - Data structures and Languages and Compilers course aided me throughout the project.

Future Work

- Outline titles can be ordered using indentation or letters. - More commands will be handled as well as different LaTeX classes. - Titles for sectioning will be numbered. - Python dictionary will be used instead of if-statements. - Website can be styled using themes selected by users. - A mobile view for mobile consumers will also be implemented.

Conclusion

- The application is a minimum viable product that is yet to become a full-fledged system capable of publishing different formats of scientific documentation. - Using its current features, authors can produce good quality books and articles.
